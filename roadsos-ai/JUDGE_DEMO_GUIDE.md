## 🚀 Quick Start (1 minute)

### Start the Application:

```bash
# Terminal 1 - Backend
cd server
npm run dev
# Should show: [server] listening on http://localhost:4000

# Terminal 2 - Frontend
cd roadsos-ai/client
npm run dev
# Should show: ready started server on 0.0.0.0:3000
```

Open your browser to `http://localhost:3000` - you'll see the full-screen smart city map with floating controls.

## 💬 Q&A for Common Questions

**Q: Is this a real product?**
A: This is a hackathon prototype that demonstrates production-ready design and architecture. It could be deployed as a real system with proper backend integration.

**Q: How does the emergency system know where to route services?**
A: The app gets your geolocation, then the backend (using location APIs) finds the nearest qualified service and calculates ETA.

**Q: Why voice-first design?**
A: During emergencies, people are stressed and injured. Voice is faster and safer than typing. It's the design philosophy of JARVIS/Tony Stark's assistant.

**Q: What makes the AI multi-mode?**
A: The backend has different system prompts for each mode. Same LLM, different instructions, making it expert in specific domains.

**Q: Can this integrate with real emergency services?**
A: Yes! The API endpoints and data structures are designed for real-world integration with 911 systems, ambulance services, hospitals, etc.
